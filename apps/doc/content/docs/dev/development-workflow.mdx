## Start the stack

```bash
bun run dev
```

## Core loop

1. Update contracts first (if API surface changes).
2. Implement API behavior.
3. Regenerate route/client artifacts when required.
4. Update web domain hooks and UI consumers.
5. Run validation gate before pushing.

## API workflow

- Contracts: `packages/contracts/api`
- Implementation: `apps/api`
- Drizzle actions:

```bash
bun run api -- db:generate
bun run api -- db:push
bun run api -- db:migrate
bun run api -- db:seed
```

## Web workflow

- Use `page.info.ts` for declarative routing definitions.
- Regenerate routes after route changes:

```bash
bun run web -- dr:build
```

- Keep API consumption in domain hooks; avoid ad-hoc raw fetch in UI components.

## Testing

```bash
bun run test
bun run test:coverage
```

## Final validation

```bash
bun run check
```
