The standard development path in this repository is container-first for consistent runtime behavior across machines.
</DocCallout>

## Prerequisites

- Bun `>= 1.3.1`
- Node.js `20+`
- Docker + Compose plugin
- Git
- A valid `.env` generated from repository templates

## Initialize project context

1. Clone the repository.
2. Generate environment variables from templates:

```bash
bun run init
```

3. Install workspace dependencies:

```bash
bun install
```

4. Optionally verify environment contract:

```bash
bun run check-env
```

## Start development

- Full stack (API + web + infra):

```bash
bun run dev
```

- Focused stack variants:

```bash
bun run dev:api
bun run dev:web
```

## URLs

- Web: http://localhost:3000
- API: http://localhost:3001 (health: /health)
- DB: Postgres 5432
- Cache: Redis 6379

## First validation cycle

Before opening a PR, run the baseline quality gate:

```bash
bun run check
```

You can also run focused checks:

```bash
bun run test
bun run lint
bun run build
```
